# Current Mode

**exploration**

We are at the start of the project. APIs are not yet stable. Breaking changes are fine. Prioritize learning what works over preserving what exists.

## Mode definitions

- **exploration** — prototyping, breaking the API is fine, no backward compatibility required
- **stabilization** — no new features, polish and bug fixes only, document the patterns that worked
- **release** — only critical fixes, every change requires extra review, all changes go through full eval suite

## Switching modes

When switching modes, update this file and note what triggered the switch. Mode changes are deliberate, not silent.
